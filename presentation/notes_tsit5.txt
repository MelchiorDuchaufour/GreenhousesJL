==============================================================================
TSIT5 — what we said, kept as a handy cheat sheet
==============================================================================

1. WHAT TSIT5 IS
----------------
Tsit5 is the **Julia / SciML name for the explicit adaptive Runge-Kutta pair
of order 5(4) published by Ch. Tsitouras in 2011**, in:

    Tsitouras Ch. (2011). "Runge-Kutta pairs of order 5(4) satisfying only
    the first quadrature condition." Computers & Mathematics with
    Applications 62(2), 770-775. DOI: 10.1016/j.camwa.2011.06.002.

"Pair of order 5(4)" means the same seven stage evaluations produce:
  - a 5th-order-accurate update (the answer we keep),
  - an embedded 4th-order update (the estimate used to gauge error).

It is the default non-stiff solver in DifferentialEquations.jl and the one
GreenhousesJL uses throughout.


2. WHY WE PICKED IT (ADVANTAGES)
--------------------------------
- **5th-order accuracy**: the local truncation error scales as h^5. Halving
  the step divides the error by 32.
- **FSAL (First-Same-As-Last)**: the seventh stage of step n is reused as
  the first stage of step n+1, so a nominally 7-stage method only costs
  six right-hand-side evaluations per step in practice. Cheapest 5(4) pair
  on the market.
- **Optimised coefficients**: Tsitouras picked (a_ij, b_i, b*_i, c_i) to
  minimise the principal 5th-order error coefficient while keeping the
  FSAL property. Gives more accuracy per function call than classical
  Dormand-Prince RK5(4).
- **Adaptive step size**: no need to pick dt. Controller automatically
  shrinks the step where dynamics are fast (sunrise, HVAC switching) and
  grows it in calm periods (steady night).
- **Dense output**: built-in 4th-order interpolation lets us plot / probe
  the solution at any time without re-integrating.
- **Fits our problem perfectly**: non-stiff, smooth, small state (see §6).

Practical speed on GreenhousesJL:
    7-day integration, Venlo greenhouse, full physics:
        ~ 4 to 30 ms  on one CPU core
        ~ 250 ms       for the OpenModelica equivalent
    i.e. 50-100x faster, for the same tolerance.


3. HOW IT WORKS (ONE PARAGRAPH)
-------------------------------
Given the current state x_n and step size h, Tsit5 performs seven stage
evaluations of the right-hand-side f(x, t):

    k_1 = f(x_n, t_n)                       # FSAL: reused from step n-1
    k_2 = f(x_n + h a_21 k_1, t_n + c_2 h)
    k_3 = f(x_n + h (a_31 k_1 + a_32 k_2), t_n + c_3 h)
    ...
    k_7 = f(x_n+1_candidate, t_n + h)       # becomes k_1 of the next step

Two different sets of weights then give:

    x_{n+1}  = x_n + h * sum_i (b_i   * k_i)     # 5th-order answer
    x*_{n+1} = x_n + h * sum_i (b*_i  * k_i)     # 4th-order estimate

The local error estimate is err = | x_{n+1} - x*_{n+1} |.

Adaptive step control, in pseudocode:

    if err > tol:
        reject step, h := h * 0.9 * (tol/err)^(1/5)   # shrink, try again
    else:
        accept step, h_next := h * 0.9 * (tol/err)^(1/5)  # grow for next

For GreenhousesJL we use reltol = 1e-6, abstol per-state (tighter for
vapor pressures). At night h can stretch to ~2 minutes; at sunrise it
contracts to a few seconds when cover, canopy and air all move together.


4. INITIAL STATE: HOW IT IS SET UP (NOT A TSIT5 CONCERN)
--------------------------------------------------------
Tsit5 does not set up the state. You give it an initial vector u0 and a
time span, and it integrates forward. The Julia pattern is:

    prob = ODEProblem(rhs!, u0, tspan, p)
    sol  = solve(prob, Tsit5(), reltol = 1e-6)

In GreenhousesJL the helper `initial_state(p::GreenhouseParams)` builds u0
consistently, in a fixed order:

    u[1]     = T_cov            (cover T)
    u[2]     = T_air            (indoor air)
    u[3]     = T_flr            (floor)
    u[4..7]  = T_wall_{N,S,E,W} (if walls enabled)
    u[next]  = T_canopy         (if crop enabled)
    ...      = C_Buf, C_Leaf, C_Stem  (crop carbon buffers)
    ...      = g_s              (dynamic stomatal conductance, if enabled)
    ...      = 12 soil layers
    ...      = VP_air           (indoor vapor pressure)
    ...      = CO2_air          (indoor CO2)
    ...      = PCM enthalpy     (if PCM enabled)

Typical cold-start defaults: 5 deg C for all temperatures, 3 deg C for
soil, VP_air from 70 % RH, CO2 at 420 ppm, stomata closed. Override with
keyword arguments on `initial_state(p; T0=..., T_soil0=...)`.

Three levels of realism:
  (1) cold-start defaults        - fastest to set up, ~12 h transient
  (2) align u0 to weather at t_0 - modest transient, typical use
  (3) spin-up run (2-7 days)     - needed when deep soil or PCM dominate
                                   the annual cycle (Brussels December
                                   benchmark uses this)

Because Tsit5 is adaptive, a wrong u0 is not catastrophic: the solver
just shrinks its step until the fast transients die out, then expands.
Spin-up is for physical consistency (slow modes equilibrated), not for
numerical stability.


5. WHY IT FITS OUR PROBLEM (NON-STIFF, SMOOTH, SMALL)
-----------------------------------------------------
- **Non-stiff**: characteristic time scales are seconds to minutes, not
  microseconds. No mode faster than ~10 s (PV cell T). An explicit
  method is stable.
- **Smooth**: f(x, t) is differentiable in every state. Discrete events
  (screen deploy, vent snap-open, CO2 injector on/off, dispatch switch)
  are handled as ODE *callbacks* that reinitialise the integrator
  cleanly, rather than as stiff switches in f itself.
- **Small**: ~25-40 state variables, plus 12 soil layers. Dense matrix
  overhead is negligible; no need for a sparse solver.
- **No algebraic constraints**: pure ODE, no DAE, no implicit equations
  to solve inside the RHS.

These are the textbook conditions where explicit adaptive Runge-Kutta
dominates: faster than implicit BDF, more accurate per evaluation than
fixed-step RK4, simpler than Rosenbrock.


6. ONE-LINE ELEVATOR PITCH
--------------------------
"Tsit5 is Julia's default 5th-order adaptive Runge-Kutta: it picks its
own step size, costs six RHS evaluations per step thanks to FSAL, and
integrates the whole greenhouse ODE in a few milliseconds per simulated
day - 50 to 100 times faster than the equivalent OpenModelica run, at
the same tolerance."
